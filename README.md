# 勤怠記録アプリ セットアップガイド

NPO法人manabo-de 向け勤怠管理PWA  
**URL:** https://fujitary.github.io/kintai2/

---

## 概要

Googleアカウントでサインインして出勤・退勤をワンタップで記録するアプリです。  
記録はクラウド（Firestore）に保存され、自動でGoogleスプレッドシートの業務日誌に書き込まれます。

---

## 初回セットアップ手順（メンバー向け）

### ① アプリにアクセス

ブラウザで以下のURLを開きます。

```
https://fujitary.github.io/kintai2/
```

**スマホの場合（推奨）：**

- iPhone：Safariで開く → 共有ボタン（↑）→「ホーム画面に追加」
- Android：Chromeで開く → 「ホーム画面に追加」バナーをタップ

ホーム画面に追加するとアプリとして使えます。

---

### ② Googleアカウントでサインイン

「Googleでサインイン」ボタンをタップして、自分のGoogleアカウントでログインします。

- 同じブラウザ・スマホでは次回から自動ログインされます
- ログアウトは上部のユーザーチップ → 「サインアウト」

---

### ③ プロフィールを設定する

初回ログイン時に「プロフィール設定」画面が表示されます。

| 項目 | 内容 |
|------|------|
| 氏名（必須） | スプレッドシートの氏名欄に入力されます |
| 役職（任意） | スプレッドシートの役職欄に入力されます |

入力後「設定を保存して開始」をタップします。

---

### ④ GAS URLを設定する

右上の「GAS」ボタンをタップして、管理者から共有されたGAS URLを貼り付けます。

```
https://script.google.com/macros/s/xxxxx/exec
```

「保存」をタップすると「GAS接続済」に変わります。

---

### ⑤ スプレッドシートIDを設定する（管理者が行う）

管理者が Firestore コンソールから各メンバーのドキュメントに以下を追加します。

| フィールド名 | タイプ | 値 |
|---|---|---|
| `spreadsheetId` | string | スプレッドシートのID |
| `spreadsheetUrl` | string | スプレッドシートのURL |

> スプレッドシートIDはURLの `/d/` と `/edit` の間の文字列です。  
> 例：`https://docs.google.com/spreadsheets/d/【ここがID】/edit`

## 日常の使い方

### 出勤する

1. 打刻タブでプロジェクトをタップして選択
2. 「🟢 出勤」ボタンをタップ
3. 経過時間がカウントアップされます

### 退勤する

1. 必要があればメモ欄に作業内容を入力
2. 「🔴 退勤」ボタンをタップ
3. スプレッドシートのその日の行に自動で書き込まれます

### 昼休憩のアラートについて

出勤〜退勤の間に **12:00〜13:00** が含まれる場合、退勤後にアラートが表示されます。  
スプレッドシートの **E列（休憩時間等）** に手動で入力してください。

## 各タブの説明

| タブ | 内容 |
|------|------|
| ⏱ 打刻 | プロジェクト選択・出勤退勤・メモ入力 |
| 📋 記録 | 月ごとの打刻履歴。✕ボタンで削除可能 |
| 📊 月次 | 合計時間・作業日数・プロジェクト別グラフ・CSVエクスポート |

---

## プロジェクトの管理

- **追加：** 「プロジェクトを追加...」に入力して「追加」ボタン
- **削除：** 「編集」ボタン → カードがぷるぷる震える → ✕ボタン → 「完了」

---

## スプレッドシートへの書き込み内容

| 列 | 内容 |
|----|------|
| C列 | 出勤時間（HH:mm） |
| D列 | 退勤時間（HH:mm） |
| L列 | プロジェクト名：メモ |

> E列（休憩時間等）・F列（総稼働時間）・K列（従事時間数）は自動計算セルのため書き込みません。

## トラブルシューティング

### スプレッドシートに反映されない

1. 右上の「GAS」ボタンをタップしてURLを確認する
2. 月次タブの「☁️ スプレッドシートへ送信」ボタンで手動送信する
3. 記録タブに「未」バッジがある場合は「🔄 未送信をGASへ再送信」をタップ

### 画面が古いまま（キャッシュ問題）

Chromeの場合：
1. F12 → アプリケーション → Service Workers
2. 「登録解除」をクリック
3. Cmd+Shift+R（Mac）/ Ctrl+Shift+R（Windows）で強制リロード

### アプリが前のバージョンに戻る

上記のキャッシュクリア手順を実施してください。

## 管理者向け情報

### 使用しているサービス

| サービス | 用途 |
|---------|------|
| Firebase Authentication | Googleログイン |
| Firestore | データ保存・リアルタイム同期 |
| Google Apps Script | スプレッドシートへの書き込みAPI |
| GitHub Pages | アプリのホスティング |

### Firestoreのデータ構造

```
users/
  {uid}/
    name: "氏名"
    role: "役職"
    gasUrl: "GASのURL"
    spreadsheetId: "スプレッドシートID"
    spreadsheetUrl: "スプレッドシートURL"
    session: null（出勤中は {projectId, startTime}）
    projects/
      {id}: { name, color, createdAt }
    records/
      {id}: { projectId, projectName, start, end, duration, memo, syncStatus }
```

### メンバーの初期設定（Firestoreへの手動追加）

新メンバーが初回ログイン後にプロフィール設定を完了しない場合、Firestoreコンソールから手動でフィールドを追加できます。

1. [console.firebase.google.com](https://console.firebase.google.com) → Firestore
2. `users` コレクション → 対象メンバーのドキュメントを開く
3. 「＋ フィールドを追加」で `name`・`role`・`spreadsheetId`・`spreadsheetUrl` を追加

### GASスクリプトの更新手順

1. [script.google.com](https://script.google.com) でプロジェクト（kintai-gas-temp）を開く
2. 新しいコードを貼り付けて保存
3. 「デプロイ」→「デプロイを管理」→ 鉛筆アイコン
4. バージョン：「新しいバージョン」を選択 → 「デプロイ」
5. URLは変わらないのでアプリの再設定は不要
